Home Media Server

What's a home media server?

And why would somebody want one?

A home media server is a centralized storage solution that allows you to store, organize, and stream digital media files such as movies, music, and photos. By setting up a media server, you can access your content from any device on your home network and, with the right configuration, even stream media remotely. Popular options like Jellyfin, Plex, and Emby provide user-friendly interfaces and various features to enhance your home entertainment experience.
While Plex and Emby have many pros associated with them, I chose to use Jellyfin for my setup. The reason was pretty simple: It's free and open source!

How the server came to be

My old roommate and I sourced an office computer off eBay, each chipping in just $10. After some setup, we installed Proxmox to run containers, turning it into a versatile home server. Now, it's our go-to hub, hosting Minecraft and Terraria servers, plus Jellyfin for streaming media. Itâ€™s been a great way to repurpose old hardware and get more value from a budget-friendly setup!

How does Jellyfin work?

Is there a super long cord running from your daily driver PC to the box sitting on the floor of the living room?

Samba Network Drives "tv shows" and "movies" Thank goodness no. There is a super cool file sharing system that allows for the creation of a network drive on a linux machine that can be accessed by windows devices called samba. This way, I can rip movie files from DVDs I own using my normal windows computer and upload the files to the linux computer running the Jellyfin server. Once the files are on the Linux machine Jellyfin is able to process the video file and pull important metadata from the internet. This includes title cards, subtitles, and cast info.

How much media can you store?

Dynamically allocated drive space The server came with 250Gb of storage on a hard drive. The speed of the hard drive isn't a huge issue (typically hard drives are MUCH slower than ssd storage), however, the capacity is. I didn't want to spend a lot of money on new storage devices, so I bought a super cheap 2Tb hard drive online and also looted my little sister's old laptop and stole her old 500Gb hard drive. With all of these drives, the system now has an ultimate capacity of 2.75Tb, which is enough to get a solid collection uploaded to the server for everyone to enjoy. Now that I've added all the drives to the computer I'll be able to upload to my heart's content, right?
Almost.
Before I can upload media to these drives, I have to ensure that the OS can locate and communicate with these drives. Additionally, I am storing all the content in two files on one container, in order to make sure these files have access to all 3 drives, the storage has to be configured on the OS side. To do this, I created volumes that the container sees as a local drive. I also configured the volumes to dynamically expand as they fill up, that way different containers (like the one running a minecraft server) can access different volumes and the storage on the drive will be dynamically allocated whenever more space is needed. Finally I can upload every video file I've ever had to the system and stream 24/7 with insanely small latency and extremely high quality.
